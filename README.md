JagDashThe Light, Decoupled, Server-Side Dashboard FrameworkJagDash is a minimalist, plugin-based dashboard framework built with Python (FastAPI) and HTMX. It delivers a high-performance workspace environment entirely driven by server-side rendered HTML fragments, eliminating complex client-side JavaScript builds, node modules, and external CDN dependencies.The framework core is strictly domain-agnostic. It provides only the foundational infrastructure—authentication, dynamic runtime profile management, a centralized capability broker, and an real-time CSS theme compilation system.All end-user panel workspaces, data analysis grids, and form actions are implemented as completely self-contained, folder-isolated plugin packages.── KEY ARCHITECTURAL FEATURES ──1. Zero-Footprint Frontend ArchitectureHTMX Native: UI components update dynamically via targeted partial swaps over standard HTTP POST/GET requests. The browser never triggers full page flushes.Local Distribution Only: Critical operational dependencies (like htmx.min.js and master styles) are packaged directly inside the core local /static/ folder. The app runs completely air-gapped without requiring cloud CDN connections.CSS Theme Injection Engine: A centralized profile engine parses user theme configurations directly into real-time CSS custom property (:root) variables, dynamically binding layouts without manual stylesheet edits.2. Hot-Reloadable Folder-Isolated Plugin SuitesAbsolute Co-location: Plugins store their backend processing scripts (plugin.py), master control panels ({plugin_name}.html), and targeted response action panels ({plugin_name}_results.html) together within an isolated folder.Dynamic Directory Browser: Users can browse the host filesystem directly from the browser navbar to select an entirely different suite of plugins. The engine instantly unloads the old state, builds a fresh PluginHost registry table, flushes Jinja2 matching caches, and mounts new routes on the fly without restarting the web server.The Hub-and-Spoke Separation Law: Plugins never cross-import or communicate directly with each other. All inter-plugin requests route through a capability broker (context.request()), ensuring that a failure in one plugin never brings down the platform or other sibling panels.── TARGET ENVIRONMENT RADAR ──Current Phase: Single-User Workspace (Highly Customizable)The current framework specification is architected for local or self-hosted deployment instances serving a single operator. The platform provides extreme customization mechanics, including multiple local profiles, dynamic dashboard renaming, custom logo asset path bindings, and custom theme overrides.Future Horizon: Multi-User Server ArchitectureThe underlying engine is designed from the ground up on top of stateless FastAPI ASGI handlers and Starlette SessionMiddleware. This ensures a seamless transition path to multi-user environments. Future architectural iterations will introduce partitioned profile databases, multi-tenant workspace registries, and role-based capability permissions.── INSTALLATION & WORKSPACE SETUP ──1.
